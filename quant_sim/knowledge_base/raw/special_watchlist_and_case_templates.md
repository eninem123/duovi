# Special Watchlist And Case Templates

## Usage boundary

- This file stores user-defined long-term monitoring themes for the research layer.
- Names, companies, and relationships listed here are research clues, not established facts.
- The agent must only use them when public disclosures, filings, ownership records, or credible reporting provide evidence.
- If evidence is weak, the answer must explicitly mark the clue as unverified and avoid presenting it as conclusion.

## Historical case templates

### Hong Kong low-price equity template

- Core question: when does a low nominal price reflect neglect rather than terminal decline?
- Compare:
  - liquidity improvement
  - policy opening
  - capital concentration
  - float structure
  - rerating trigger

### Japan property supply-demand template

- Core question: when does supply-demand change create structural repricing instead of a short-term bounce?
- Compare:
  - demographic or migration shift
  - financing cost
  - vacancy trend
  - foreign capital access
  - redevelopment timing

### Old factory redevelopment template

- Core question: when does cheap rent plus early pilot policy create hidden convexity?
- Compare:
  - land scarcity
  - policy continuity
  - first-wave tenant quality
  - traffic and tourism rerouting
  - asset valuation gap versus the city core

### Southeast Asia opening-up template

- Core question: when does foreign ownership relaxation create an early-cycle property or equity opportunity?
- Compare:
  - price gap versus mature peers
  - legal protection
  - currency risk
  - exit liquidity
  - infrastructure spillover

## Response protocol for this material

- When the user asks for cross-domain opportunity discovery, prioritize this file and similar research notes.
- Output should separate:
  - ignored details
  - benefit logic
  - verification path
  - risk flags
  - next action
- Do not convert watchlist clues into trade recommendations without real-time market and rule checks.
