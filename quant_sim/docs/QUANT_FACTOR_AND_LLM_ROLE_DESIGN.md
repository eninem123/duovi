# 量化因子体系与 LLM 职责边界设计方案

## 1. 引言

本设计方案旨在优化 `duovi` 交易系统的策略逻辑，解决当前系统过度依赖 LLM 主观判断“综合胜率”的问题。通过引入可验证的量化因子，并明确 LLM 在决策流程中的辅助角色，构建一个“量化因子支撑 + 逻辑分析辅助”的复合决策体系。核心目标是提升决策的客观性、可解释性和稳定性。

## 2. 现状分析与问题

当前 `duovi` 系统的买入决策主要依赖 LLM 输出的 `win_rate_confidence`（综合胜率）和 `dimension_scores`（数据架构、知识库、博弈心理、趋势动能）进行判断。存在以下问题：

*   **LLM 职责边界模糊**：LLM 直接承担了“买不买”的决策，而非其擅长的非结构化信息解读和定性分析。
*   **量化因子缺失**：缺乏明确、可验证的量化因子（如动量、估值、流动性、技术面）作为硬性筛选条件。
*   **胜率阈值主观**：`win_rate_threshold: 0.75` 等参数缺乏量化依据，易导致策略不稳定。
*   **评分维度抽象**：`dimension_scores` 的 0-25 分评分机制难以量化和验证，LLM 输出的可靠性存疑。

## 3. 优化目标

1.  **引入多维度量化因子**：集成动量、估值、流动性、技术面等因子，作为股票筛选的第一道门槛。
2.  **重塑 LLM 角色**：将 LLM 的核心职责限定为解读财报、研报、新闻舆情等非结构化信息，识别政策催化、产业链逻辑变化等定性因素，为通过量化筛选的股票提供深度背景分析和风险提示。
3.  **构建复合决策流程**：形成“量化因子初筛 → LLM 深度解读 → 综合决策”的流程，提升决策的科学性和鲁棒性。
4.  **移除主观胜率判断**：废弃 LLM 直接输出的 `win_rate_confidence` 和 `dimension_scores` 作为买入依据。

## 4. 方案设计

### 4.1. 核心决策流程重构

新的决策流程将分为以下几个阶段：

1.  **数据获取与预处理**：获取市场行情、财务数据、交易数据等，为量化因子计算做准备。
2.  **量化因子筛选**：根据 `config.yaml` 中定义的动量、估值、流动性、技术面等因子，对股票池进行初步筛选，生成候选股票列表。
3.  **LLM 深度解读**：对通过量化筛选的候选股票，LLM 将分析其相关的非结构化信息（财报、研报、新闻等），输出定性分析报告，包括但不限于：
    *   **基本面解读**：公司业务、行业地位、竞争优势、财报亮点/风险。
    *   **催化剂分析**：政策支持、技术突破、市场事件等潜在驱动因素。
    *   **风险提示**：宏观风险、行业风险、公司特定风险。
    *   **市场情绪与博弈**：结合新闻舆情分析市场对该股票的关注度和情绪。
4.  **综合决策**：结合量化因子的硬性过滤结果和 LLM 的定性分析报告，由系统（或人工复核）做出最终的买入/观望决策。

### 4.2. 量化因子体系

根据用户建议，引入以下量化因子，并将其配置化：

| 因子类别 | 因子名称 | 配置参数 | 描述 | 数据来源 | 接入模块 |
| :------- | :------- | :------- | :--- | :------- | :------- |
| **动量** | 相对强度动量 | `lookback_days` (5, 10, 20), `min_rank_percentile` (70) | 筛选在指定周期内表现优异的股票，只买动量排名前 30% 的股票。 | 历史行情数据 | `quant_factors.py` (新增) |
| **估值** | 市盈率 (PE) | `pe_percentile_max` (60) | 筛选估值合理的股票，PE 不超过历史 60 分位。 | 财务数据 | `quant_factors.py` (新增) |
|          | 市净率 (PB) | `pb_percentile_max` (50) | 筛选估值合理的股票，PB 不超过历史 50 分位。 | 财务数据 | `quant_factors.py` (新增) |
| **流动性** | 20 日均成交额 | `min_avg_turnover_20d` (50,000,000) | 确保股票具有足够的流动性，日均成交额大于 5000 万。 | 交易数据 | `quant_factors.py` (新增) |
|          | 买卖价差 | `max_bid_ask_spread_pct` (0.005) | 控制交易成本，买卖价差不超过 0.5%。 | 实时行情数据 | `quant_factors.py` (新增) |
| **技术面** | 均线排列 | `ma_trend_aligned` (true) | 判断股票处于多头趋势，例如 5 日均线 > 10 日均线 > 20 日均线。 | 历史行情数据 | `quant_factors.py` (新增) |
|          | 量比 | `volume_ratio_min` (1.5) | 筛选当日交易活跃的股票，量比大于 1.5。 | 实时交易数据 | `quant_factors.py` (新增) |

### 4.3. LLM 职责重塑

LLM 的角色将从“决策者”转变为“信息解读与分析师”。具体调整如下：

*   **输入**：接收通过量化因子筛选的股票列表，以及相关的市场行情、新闻、研报摘要等非结构化信息。
*   **输出**：生成一份结构化的定性分析报告，不再包含 `win_rate_confidence` 和 `dimension_scores`。报告应侧重于：
    *   **核心逻辑**：该股票的基本面、行业前景、竞争优势。
    *   **催化剂**：短期或中长期可能驱动股价上涨的事件或政策。
    *   **风险点**：潜在的宏观、行业或公司特定风险。
    *   **情绪解读**：市场对该股票的关注度、情绪倾向。
*   **提示词优化**：`agent.execution_prompt` 将被修改，移除对 `win_rate_confidence` 和 `dimension_scores` 的要求，转而引导 LLM 专注于上述定性分析。

### 4.4. 模块调整

1.  **新增 `quant_factors.py` 模块**：
    *   负责从数据源获取原始数据。
    *   实现动量、估值、流动性、技术面等量化因子的计算逻辑。
    *   提供一个 `screen_stocks(universe, config)` 函数，根据配置的因子条件筛选股票。
2.  **修改 `risk_gate.py`**：
    *   在 `buy_blocked_reason` 函数中，增加对量化因子筛选结果的检查。只有通过量化因子筛选的股票才能进入后续的 LLM 分析阶段。
    *   移除 `win_rate_threshold` 相关的逻辑。
3.  **修改 `mcp_agent.py`**：
    *   `make_decision` 函数将不再直接依赖 LLM 的 `win_rate_confidence`。它将首先调用 `quant_factors.screen_stocks` 进行初步筛选。
    *   LLM 的调用将调整为对通过量化筛选的股票进行定性分析，其输出将作为决策的辅助信息。
    *   `decision_json_instruction` 将被修改，以反映新的 LLM 输出结构。
4.  **修改 `main.py`**：
    *   调整 `run_decision_cycle` 的流程，使其首先执行量化因子筛选，然后将筛选结果传递给 LLM 进行分析。
    *   最终的买入决策将综合量化因子结果和 LLM 的定性分析。
5.  **修改 `config.yaml`**：
    *   已移除 `trading.win_rate_threshold`。
    *   已新增 `factors` 配置节，包含动量、估值、流动性、技术面等因子的具体参数。

## 5. 风险与挑战

*   **数据获取**：量化因子依赖准确、实时的市场数据和财务数据。需要确保数据源的稳定性和可靠性。
*   **因子参数优化**：量化因子的具体参数（如 `lookback_days`、`min_rank_percentile`）需要通过回测和实盘验证进行持续优化。
*   **LLM 幻觉**：尽管 LLM 职责已调整，仍需警惕其在定性分析中可能出现的“幻觉”或不准确信息，需要引入交叉验证机制。
*   **性能**：引入更多量化因子计算和 LLM 调用可能增加系统延迟，需要关注性能优化。

## 6. 总结

本设计方案通过引入多维度量化因子和重塑 LLM 职责，旨在构建一个更健壮、更科学的 `duovi` 交易决策系统。这将使系统能够更好地结合量化分析的客观性和 LLM 的定性洞察力，从而提升整体策略的有效性。
